## Meta Description
Writing your resume in Markdown keeps it version-controllable and ATS-friendly — but most job applications still want a .docx. Here's how to get both.

## Key points to cover
- Address the real reason people search this: many ATS (applicant tracking system) guides recommend simple, single-column, heading-based resumes — which happens to be exactly what Markdown naturally produces, making this a genuinely good-fit use case rather than a stretch.
- Recommend against trying to replicate complex two-column resume templates in Markdown — be honest that Markdown's simplicity is the advantage here, not a limitation to work around.
- Suggest a final formatting pass in Word (font choice, margins, spacing) since a converted document is a strong structural starting point, not a finished polished resume.
